“unity深度纹理重建世界坐标”

发表于 2022-03-29 分类于 unity

从深度纹理中重构世界坐标

坐标系中的一个顶点坐标可以通过它相对于另一个顶点的坐标偏移量来求得。重构像素的世界坐标也是基于这样的思想——>我们只需要知道摄像机在世界空间下的位置，以及世界空间下该像素相对于摄像机的偏移量，把它们相加就可以得到该像素的世界坐标

float4 worldPos = _WorldSpaceCameraPos + linearDepth * interpolatedRay

其中_WorldSpaceCameraPos是摄像机在世界空间下的位置（可以由Unity的内置变量直接访问得到），inearDepth * interpolatedRay可以计算得到该像素相对于摄像机的偏移量，linearDepth是由深度纹理得到的线性深度值，interpolatedRay是由顶点着色器输出并差值后得到的射线。

interpolatedRay来源于对近裁剪平面的4个脚的某个特定向量的插值，这个向量包含了它们到摄像机的方向和距离信息，我们可以利用摄像机的近裁剪平面距离、 FOY、横纵比计算而得

为方便计算，我们可以先计算两个向量——toTop和toRight它们是起点位于近剪裁平面中心，分别指向摄像机正上方和正右方的向量：
halfHeight = Near * tan(FOV/2) //Near是金剪裁平面的距离，FOV是竖直方向的视角范围
toTop = camera.up * halfHeight
toRight = camera.right * halfHeight*aspect

得到了这两个辅助向量之后，我们就可以计算4个角相对于摄像机的方向了
TL=camera.forward * Near + toTop - toRight
TR=camera.forward * Near + toTop + toRight
BL=camera.forward * Near - toTop - toRight
BR=camera.forward * Near - toTop + toRight
注意，上面求得的4个向量不仅包含了方向信息，它们的模对应了4个点到摄像机的空间距离。由于我们得到的线性深度值并非是点到摄像机的欧式距离，而是在z方向上的距离，因此，我们不能直接使用深度值和4个角的单位方向的乘积来计算它们到摄像机的偏移量。如图13.7所示，不难由相似三角形得出：depth / dist = Near / | TL |。 即：我们需要的TL距离摄像机的欧式距离dist：
dist = | TL | * depth / Near
由于4个点相互对称，因此其他3个向量的模和TL相等，即我们可以使用同一个因子和单位向量相乘，得到它们对应的向量值：scale = | TL | / | Near |
由于我们的后处理实际上就是渲染了一个Quad，上下左右四个顶点，把这个射线传递给pixel阶段时，就会自动进行插值计算，也就是说在顶点阶段的方向值到pixel阶段就变成了逐像素的射线方向。

opencascade gltf

发表于 2022-03-25 更新于 2022-04-01 分类于 opencascade

关于OPENCASCADE中的GLTF模型的导出

glTF格式本质上是一个JSON文件。 这一文件描述了整个3D场景的内容。 它包含了对场景结构进行描述的场景图。 场景中的3D对象通过场景结点引用网格进行定义。
glTF详情可参考：https://blog.csdn.net/qq_31709249/article/details/86477520

在opencascade中，实现gltf导出的类为RWGltf_CafWriter，欲使用此类，需要使用到XCAFDoc相关内容。本人理解中的gltf在opencascade中的导出即是将想要导出的内容通过各种方法操作填充到一个TDocStd_Document中，为此需要准备的XCAFDoc操作类基本有
1.XCAFDoc_ShapeTool——用于添加Shape并进行Shape相关操作
2.XCAFDoc_ColorTool——用于进行颜色相关操作
3.XCAFDoc_MaterialTool——用于定义物体的材质等等（注意：此处的材质指的是密度等，在视觉方面无影响）
4.XCAFDoc_VisMaterialTool——定义材质属性（可用的有标准光照属性与pbr光照属性）

本文重点介绍如何设置一般光照相关的属性并将其赋予TopoDS_Shape

首先初始化一个XCAFDoc_VisMaterial类mat，此mat将会用于定义材质属性与渲染模式以及最终赋予需要的TopoDS_Shape。
然后我们需要一个XCAFDoc_VisMaterialCommon类mac，及基本的材质属性（非pbr），我们将在此类中进行texture、diffusecolor等等的定义。
需要注意的是，在定义XCAFDoc_VisMaterialCommon时，一定一定一定要确保IsDefined被设置为TRUE（默认为FALSE），否则一切对此类的操作都不会生效。
完成mac的参数设置后，我们将mac赋予mat：mat->SetCommonMaterial(mac);
最后，我们使用XCAFDoc_VisMaterialTool将设置好的mat赋给需要的TopoDS_Shape：VT->SetShapeMaterial(myobject.shape, VT->AddMaterial(mat1, “somemessage”))
至此，我们就完成了对需要输出的shape的材质、贴图等的设置。

二叉树小结

发表于 2021-12-04 分类于 c++

树 是一种经常用到的数据结构，用来模拟具有树状结构性质的数据集合。

树里的每一个节点有一个值和一个包含所有子节点的列表。从图的观点来看，树也可视为一个拥有N 个节点和N-1 条边的一个有向无环图。

二叉树是一种更为典型的树状结构。如它名字所描述的那样，二叉树是每个节点最多有两个子树的树结构，通常子树被称作“左子树”和“右子树”。

前序遍历（根左右）
前序遍历首先访问根节点，然后遍历左子树，最后遍历右子树。
中序遍历（左根右）
中序遍历是先遍历左子树，然后访问根节点，然后遍历右子树。
后序遍历（左右根）
后序遍历是先遍历左子树，然后遍历右子树，最后访问树的根节点。
值得注意的是，当你删除树中的节点时，删除过程将按照后序遍历的顺序进行。 也就是说，当你删除一个节点时，你将首先删除它的左节点和它的右边的节点，然后再删除节点本身。
另外，后序在数学表达中被广泛使用。 编写程序来解析后缀表示法更为容易。
